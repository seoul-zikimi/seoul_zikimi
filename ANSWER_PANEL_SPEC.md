# 기능 스펙 — 정답 패널 인터랙티브 3D 뷰 (카메라 조작 공유)

> 소속: "정답 안내 UX" 작업의 일부. 정답 패널을 **정적 이미지가 아니라 돌려보고 확대할 수 있는 3D 뷰**로 만든다.
> 원칙: 정답 패널에 포커스되면 **플레이어 카메라 조작과 동일한 입력**(우클릭 드래그 회전 / 스크롤 줌)이 정답 패널 카메라에 적용된다.

---

## 1. 한 줄 정의
정답 건물을 보여주는 패널에 포커스가 가면, 플레이어 뷰를 돌리던 것과 **똑같은 조작감**으로 정답 모델을 회전·확대해서 비교할 수 있다.

## 2. 조작 매핑 (기획서 2.2 그대로 재사용)
| 입력 | 플레이어 카메라 | **정답 패널 카메라 (포커스 시)** |
|---|---|---|
| 우클릭 + 드래그 | 카메라 회전 | **정답 모델 오빗(궤도) 회전** |
| 마우스 스크롤 ↑/↓ | 확대/축소 | **정답 모델 확대/축소** |
| 기본 각도 | 쿼터뷰(위·오른쪽 ~30°) | **동일 쿼터뷰** — 플레이어 시점과 같은 기준으로 비교 가능하게 |

→ 두 카메라가 **같은 조작 로직**을 쓰는 게 핵심. 손이 헷갈리지 않음.

## 3. 카메라 차이 (플레이어 vs 정답)
| | 플레이어 카메라 | 정답 패널 카메라 |
|---|---|---|
| 추적 | 플레이어를 딜레이 추적(damped follow) | 추적 없음. **정답 모델 중심을 오빗** |
| 대상 | 메인 씬 | 정답 모델 전용(분리 렌더) |
| 줌/회전 한계 | 플레이 그리드 기준 | **정답 모델 바운드 기준**으로 클램프 |
| 상호 영향 | — | 정답 조작은 플레이어 카메라에 영향 X (완전 독립) |

## 4. 포커스(입력 라우팅) 설계 — 핵심
하나의 마우스 입력을 **현재 포커스 대상**(플레이어 / 정답 패널)에 따라 분배한다.

- **권장 트리거: 위치 기반 라우팅.**
  - 마우스 커서가 **정답 패널 영역 안**에 있을 때의 우클릭 드래그/스크롤 → 정답 카메라로.
  - 패널 밖이면 → 평소대로 플레이어 카메라로.
  - 마찰 0 (별도 모드 진입/해제 동작 불필요), 실수로 캐릭터 카메라 안 돌아감.
- 입력 소스는 하나, `InputRouter`(또는 focus state)가 분기. 양쪽 카메라는 동일 컴포넌트 공유.

## 5. 렌더링 구현 방향
- 정답 모델을 메인 씬과 **분리된 위치/레이어**에 인스턴스화.
- **전용 카메라 + RenderTexture**로 그 모델만 찍어서 패널 UI(RawImage)에 출력.
- 정답 모델은 인게임 빌드 → 저장한 `MapAnswerData`로부터 구성(같은 프리팹 재사용).

## 6. 코드 재사용 (LevelUpYourCode 스타일)
- 플레이어 카메라의 **오빗 + 줌 로직을 별도 컴포넌트로 추출**(예: `CameraOrbitController`)해서 플레이어 카메라와 정답 패널 카메라가 **둘 다 공유**.
  - 코드 중복 제거 + 조작감 일관성 자동 보장.
  - 입력(Input)과 카메라 동작(Behaviour) 분리. 라우팅만 갈아끼움.

## 7. 멀티플레이어
- 정답 패널 조작은 **완전 로컬.** 각 플레이어가 자기 화면에서 알아서 돌려봄.
- **동기화 X → 네트워크 트래픽 0.** 서버 권위 대상 아님.

## 8. 결정 필요 (빨간 글씨)
1. **정답 패널 상시 표시 vs 토글** — 작은 미니뷰 항상 띄워두고, 클릭하면 큰 모달로 확대해서 돌려보기? 아니면 처음부터 고정 패널? (권장: 작은 상시 미니뷰 + 확대 토글)
2. **포커스 해제 시 카메라 리셋 여부** — 정답 회전/줌을 다음에 볼 때 기본 쿼터뷰로 리셋할지, 마지막 상태 유지할지. (권장: 유지 — 다시 맞출 필요 없어 편함)
3. **TAB 공정 표시와의 관계** — 정답 패널을 돌려보는 동안에도 TAB 요구공정 오버레이를 같이 보여줄지.

---

## 9. 완료 기준 (DoD)
- 정답 패널 위에서 우클릭 드래그 → 정답 모델만 회전, 캐릭터 카메라는 안 움직임.
- 정답 패널 위에서 스크롤 → 정답 모델만 줌, 플레이 화면 줌 영향 없음.
- 패널 밖에서는 기존 플레이어 카메라 조작 그대로 동작.
- 회전/줌 로직이 양쪽에서 **동일 컴포넌트** 통해 동작(중복 코드 없음).
- 멀티에서 각자 따로 돌려봐도 동기화 트래픽 0.
