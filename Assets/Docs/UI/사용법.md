# UIManager 사용설명서

UIManager는 게임의 모든 UI 화면을 한 곳에서 관리하는 매니저입니다.  
HUD / Popup / System 세 가지 레이어로 나뉘어져 있고, 각각 열고 닫는 방식이 다릅니다.

> 📄 이 문서의 구성
> 1. 레이어 구조 이해하기
> 2. BootstrapScene에 싱글톤 배치하기
> 3. 새 UI 화면 만들기 (3단계)
> 4. Bind / BindEvent 상세 규칙
> 5. UISlot 사용법
> 6. **[참고]** GameManager 호출 흐름
> 7. **[참고]** 예상 UI 화면 목록

## 1. 레이어 구조 이해하기

UIManager는 UI 화면을 **3개의 레이어**로 분리합니다.  
레이어마다 Sorting Order가 달라서 항상 올바른 순서로 화면 위에 쌓입니다.

| 레이어 | Sorting Order | 베이스 클래스 | 언제 쓰나요? |
|--------|:---:|---|---|
| **HUD** | 10 | `UIHUD` | 게임 중 항상 떠있는 UI (타이머, 점수판, 재료 카탈로그 등) |
| **Popup** | 30 | `UIPopup` | 특정 상황에 열리는 팝업 (채점 결과, 보상, 건물 선택 등) |
| **System** | 100 | `UIPopup` | 무조건 제일 위에 떠야 하는 UI (로딩 화면, 연결 끊김 알림 등) |

### HUD vs Popup, 뭐가 다른가요?

**HUD**는 한 번 만들어지면 계속 재사용됩니다.  
`ShowHUDUI<T>()`를 두 번 호출해도 인스턴스는 하나뿐이고, 두 번째 호출 시엔 그냥 `SetActive(true)`로 다시 보여줍니다.  
→ 타이머처럼 게임 내내 쓰이는 UI에 적합합니다.

**Popup**은 열 때마다 새 인스턴스를 만들고, 스택(Stack) 구조로 관리됩니다.  
가장 나중에 열린 팝업이 가장 먼저 닫히는 LIFO 방식입니다.  
→ 채점 결과처럼 일회성으로 열었다 닫는 UI에 적합합니다.

## 2. BootstrapScene에 싱글톤 배치하기

`UIManager`는 게임 내내 살아있어야 하는 매니저입니다.  
그래서 **BootstrapScene에만 배치하고**, `DontDestroyOnLoad`를 통해 씬이 바뀌어도 유지되게 합니다.  
(`GameManager`, `SoundManager`도 같은 방식으로 BootstrapScene에 둡니다.)

### 배치 순서

**① BootstrapScene을 엽니다.**

**② UIManager 오브젝트를 만듭니다.**
```
Hierarchy 빈 곳 우클릭 → Create Empty → 이름을 "UIManager"로 변경
```

**③ UIManager 스크립트를 붙입니다.**
```
Inspector → Add Component → "UIManager" 검색 → 선택
```

**④ EventSystem이 있는지 확인합니다.**  
EventSystem이 없으면 버튼 클릭 등 UI 이벤트가 전혀 작동하지 않습니다.
```
이미 있다면 → OK
없다면 → 상단 메뉴 GameObject > UI > Event System 으로 추가
```

**⑤ GameManager, SoundManager도 나중에 같은 방식으로 추가합니다.**

완성된 BootstrapScene의 Hierarchy:
```
BootstrapScene
├── UIManager          ← UIManager.cs 붙어있음
├── GameManager        ← 나중에 추가
├── SoundManager       ← 나중에 추가
└── EventSystem        ← Unity 기본 제공
```

> ⚠️ **BootstrapScene은 빌드에서 index 0이어야 합니다.**  
> `File > Build Settings`를 열어서 BootstrapScene이 목록 맨 위에 있는지 확인하세요.  
> 그래야 게임 시작 시 가장 먼저 로드되어 싱글톤이 제대로 만들어집니다.

## 3. 새 UI 화면 만들기

새 UI 화면을 추가할 때는 항상 **3단계** 를 거칩니다.  
아래 예시를 따라하면 어떤 UI든 동일한 패턴으로 만들 수 있습니다.

---

### Step 1 — C# 클래스 작성

`Assets/Scripts/UI/HUD/` 또는 `Assets/Scripts/UI/Popup/` 에 새 스크립트를 만듭니다.

#### HUD 예시 — `TimerHUD.cs`

```csharp
public class TimerHUD : UIHUD
{
    // 프리팹에서 찾을 자식 오브젝트 이름을 enum으로 정의합니다.
    // ⚠️ enum 이름과 자식 오브젝트 이름이 대소문자까지 완전히 일치해야 합니다.
    enum Texts   { Txt_Timer, Txt_Score }
    enum Buttons { Btn_End }

    public override void Init()
    {
        // Bind를 호출하면 enum 이름으로 자식 오브젝트를 자동으로 찾아 캐싱합니다.
        Bind<TextMeshProUGUI>(typeof(Texts));
        Bind<Button>(typeof(Buttons));

        // 버튼 클릭 이벤트 연결
        BindEvent(Get<Button>((int)Buttons.Btn_End).gameObject, _ => OnEndClicked());
    }

    // 외부에서 타이머 값을 업데이트할 때 호출합니다.
    public void SetTimer(float seconds)
    {
        Get<TextMeshProUGUI>((int)Texts.Txt_Timer).text =
            $"{(int)seconds / 60:00}:{(int)seconds % 60:00}";
    }

    void OnEndClicked()
    {
        // 건축 종료 요청 로직
    }
}
```

#### Popup 예시 — `ScoringPopup.cs`

```csharp
public class ScoringPopup : UIPopup
{
    enum Texts   { Txt_TotalScore, Txt_Breakdown }
    enum Buttons { Btn_Confirm }

    public override void Init()
    {
        Bind<TextMeshProUGUI>(typeof(Texts));
        Bind<Button>(typeof(Buttons));

        // 확인 버튼을 누르면 이 팝업을 닫습니다.
        BindEvent(Get<Button>((int)Buttons.Btn_Confirm).gameObject,
            _ => UIManager.Instance.ClosePopupUI(this));
    }

    // 채점 결과를 받아서 화면에 표시합니다.
    public void SetScore(int total, string breakdown)
    {
        Get<TextMeshProUGUI>((int)Texts.Txt_TotalScore).text = $"{total}점";
        Get<TextMeshProUGUI>((int)Texts.Txt_Breakdown).text  = breakdown;
    }
}
```

### Step 2 — 프리팹 만들기

Unity Editor에서 UI 프리팹을 만들고, 아래 경로에 저장합니다.

| 레이어 | 저장 경로 |
|--------|----------|
| HUD | `Assets/Resources/UI/HUD/TimerHUD` |
| Popup | `Assets/Resources/UI/Popup/ScoringPopup` |
| System | `Assets/Resources/UI/System/LoadingSystemUI` |

> 📌 **프리팹 이름 = 클래스 이름** 이어야 합니다.  
> UIManager가 클래스 이름으로 Resources.Load를 하기 때문입니다.  
> 예: `TimerHUD.cs` → 프리팹 이름 `TimerHUD`

프리팹 안에 자식 오브젝트를 만들 때, **이름을 enum과 똑같이** 지어야 합니다.

```
TimerHUD (프리팹 루트)
├── Txt_Timer     ← enum Texts.Txt_Timer 와 이름 일치
├── Txt_Score     ← enum Texts.Txt_Score 와 이름 일치
└── Btn_End       ← enum Buttons.Btn_End 와 이름 일치
```

> ⚠️ **대소문자도 정확히 일치해야 합니다.**  
> `Txt_timer` 처럼 소문자로 쓰면 Bind가 실패합니다.

---

### Step 3 — 코드에서 호출하기

#### HUD 호출

```csharp
// HUD 열기 — 처음엔 인스턴스를 만들고, 이후엔 기존 인스턴스를 재사용합니다.
UIManager.Instance.ShowHUDUI<TimerHUD>().SetTimer(300f);

// HUD 숨기기 — 인스턴스는 유지되고 화면에서만 사라집니다. (SetActive false)
UIManager.Instance.HideHUDUI<TimerHUD>();

// TAB 키를 누르는 동안만 보여주는 패턴 (ProcessGuideHUD 등에 활용)
if (Input.GetKeyDown(KeyCode.Tab)) UIManager.Instance.ShowHUDUI<ProcessGuideHUD>();
if (Input.GetKeyUp(KeyCode.Tab))   UIManager.Instance.HideHUDUI<ProcessGuideHUD>();
```

#### Popup 호출

```csharp
// 팝업 열기 — 열 때마다 새 인스턴스가 만들어집니다.
UIManager.Instance.ShowPopupUI<ScoringPopup>().SetScore(9500, "배치 +8000\n공정 +1500");

// 가장 위에 있는 팝업 닫기
UIManager.Instance.ClosePopupUI();

// 특정 팝업 닫기 (그 팝업이 지금 맨 위에 있을 때만 작동합니다)
UIManager.Instance.ClosePopupUI(this);

// 모든 팝업 한 번에 닫기 (채점 시작 전 등에 활용)
UIManager.Instance.CloseAllPopupUI();
```

#### System UI 호출

```csharp
// 씬 전환 시 로딩 화면 띄우기
UIManager.Instance.ShowSystemUI<LoadingSystemUI>();

// 로딩 완료 후 닫기
UIManager.Instance.CloseSystemUI();
```

## 4. Bind / BindEvent 상세 규칙

### Bind란?

`Bind<T>(typeof(Enum))`는 프리팹의 자식 오브젝트를 **enum 이름으로 자동으로 찾아서 캐싱**하는 기능입니다.  
덕분에 `GameObject.Find()`를 직접 쓸 필요 없이, `Get<T>(idx)`로 간편하게 컴포넌트를 가져올 수 있습니다.

```csharp
// Bind 후 컴포넌트 가져오는 방법
Get<TextMeshProUGUI>((int)Texts.Txt_Timer)   // enum 값을 int로 캐스팅해서 인덱스로 사용
Get<Button>((int)Buttons.Btn_End)
Get<Image>((int)Images.Img_Icon)
```

### 규칙 정리

| 규칙 | 설명 |
|------|------|
| `Bind`는 `Init()` 안에서만 | `Init()`은 UIManager가 UI를 만들 때 자동으로 한 번 호출합니다 |
| enum 이름 = 자식 오브젝트 이름 | **대소문자까지 완전히 일치**해야 합니다 |
| `Get<T>((int)EnumValue)` | enum 값을 int로 캐스팅해서 인덱스로 사용합니다 |
| BindEvent type 기본값 | 기본값은 `"Click"`. 필요시 `"Drag"`, `"BeginDrag"`, `"EndDrag"`, `"Drop"` 사용 가능 |

### Bind가 실패했을 때

Console에 이런 메시지가 뜬다면 Bind가 실패한 것입니다.
```
Failed to bind(Txt_Timer)
```
**체크리스트:**
- [ ] 프리팹 안에 `Txt_Timer`라는 이름의 자식 오브젝트가 있는지 확인
- [ ] 이름의 대소문자가 enum과 완전히 일치하는지 확인 (`Txt_timer` ❌ → `Txt_Timer` ✅)
- [ ] 해당 자식 오브젝트에 Bind하려는 컴포넌트(`TextMeshProUGUI` 등)가 붙어있는지 확인

## 5. UISlot 사용법 (반복되는 목록 항목)

재료 카탈로그처럼 **같은 모양의 항목이 반복될 때**는 `UISlot`을 사용합니다.  
`UIHUD`, `UIPopup`과 사용 방법이 동일하지만, UIManager가 직접 관리하지 않고 부모 HUD/Popup이 직접 인스턴스화해서 씁니다.

```csharp
// 슬롯 클래스 작성
public class MaterialSlot : UISlot
{
    enum Images  { Img_Icon }
    enum Texts   { Txt_Count }
    enum Buttons { Btn_Select }

    public override void Init()
    {
        Bind<Image>(typeof(Images));
        Bind<TextMeshProUGUI>(typeof(Texts));
        Bind<Button>(typeof(Buttons));
    }

    public void SetData(Sprite icon, int count)
    {
        Get<Image>((int)Images.Img_Icon).sprite = icon;
        Get<TextMeshProUGUI>((int)Texts.Txt_Count).text = $"x{count}";
    }
}
```

```csharp
// MaterialCatalogHUD 내부에서 슬롯을 만드는 방법
var slot = Instantiate(slotPrefab).GetComponent<MaterialSlot>();
slot.Init();   // ⚠️ UISlot은 UIManager가 Init을 호출하지 않으므로 직접 호출해야 합니다
slot.SetData(icon, count);
slot.transform.SetParent(listContainer, false);
```

## [참고] GameManager UI 호출 흐름

> 기획서 기반 예상 흐름입니다. GameManager 구현 시 참고용으로 활용하세요.

| 게임 상태 | UIManager 호출 |
|----------|----------------|

## [참고] 예상 UI 화면 목록 (기획서 기반)

> 아직 확정된 목록이 아닙니다. 기획이 구체화되면 추가·변경될 수 있습니다.

| 클래스 | 기능 | 메모 |

| `ConfirmPopup` | 확인/취소 범용 다이얼로그 | 여러 곳에서 재사용 가능하게 설계 권장 |

### System (항상 최상단)

| 클래스 | 기능 | 메모 |
|--------|------|------|
| `LoadingSystemUI` | 씬 전환 중 로딩 화면 | Popup 위를 덮음 |
| `DisconnectPopup` | 멀티 연결 끊김 알림 | |
| `ErrorPopup` | 서버 에러 / 강제 종료 알림 | |

## [참고] Addressables로 전환하고 싶다면

현재는 `Resources.Load` 방식을 사용하고 있습니다.  
나중에 Addressables로 바꾸고 싶다면 `UIManager.cs`에서 딱 한 줄만 바꾸면 됩니다.

```csharp
// 현재 (Resources)
IAssetProvider _loader = new ResourcesProvider();

// 전환 후 (Addressables)
IAssetProvider _loader = new AddressablesProvider();  // AddressablesProvider 별도 구현 필요
```

나머지 코드는 전혀 바꾸지 않아도 됩니다.
