# 블록 · 맵 제작 가이드 (기획자용)

안녕하세요! 이 문서 하나면 **블록 만들기 + 맵(정답) 만들기**를 다 할 수 있어요.

이 게임은 **정답 건물**을 보고 친구들과 똑같이 짓는 협동 게임이에요. 기획자가 만들 건 딱 두 가지예요:

- 🧱 **블록(재료)** — 건물에 쓰는 3D 조각이에요. (NC VARCO AI로 제작)
- 🗺️ **맵(정답)** — "이렇게 지어봐!" 하는 목표 배치예요. → `MapAnswerData` 파일로 저장돼요.

> 💡 **용어 미리보기** — Unity 화면의 패널들이에요:
> **Hierarchy**(씬 안 오브젝트 목록) · **Project**(에셋/파일 목록) · **Inspector**(고른 것의 속성) · **Scene**(3D 편집 화면) · **Console**(로그/오류).

---

## 🚀 빠른 시작 — 기획자가 하는 건 3가지뿐

> 이 3개만 하시면 돼요. 나머지(주문·운반·배치·공정·붕괴·채점·멀티·재시작)는 **이미 다 만들어져 있어요.**

1. **블록 만들기** — VARCO AI로 3D 블록을 뽑아 → 프리팹으로 만들고 → 등록해요. *(→ §2, §3)*
2. **맵(정답) 만들기** — Autotiles3D에서 칸을 칠해 정답을 그리고 → 버튼 한 번으로 내보내요. *(→ §4)*
3. **게임에 등록** — 만든 정답들을 게임 씬의 `GridManager` → `Answers`에 넣어요. *(→ §5)*

처음이시면 **§0 → §4 → §5** 순서로 한 번 따라 해보시고, 진짜 블록을 만들 땐 §2·§3을 보세요.

### ✅ 지금 되는 것
- 그리드에 블록 **배치** / **공정**(고정·페인트) / **채점** (위치가 정답과 같으면 점수)
- **멀티플레이**(서버 기준으로 모두 같은 화면) — 호스트+참가자 동기화
- 재료 **주문(물류창고) → 배송 → 주워서 → 배치** / 버리거나 부순 재료는 **바닥에 떨어져 다시 주움**
- **무너짐**: 미고정 블록은 부딪히거나 위에 쌓으면 무너짐 + 떨어지는 "노답중력" 연출
- **게임 한 판**: 타이머 → 전원이 동의하면 건축 종료 → 점수 → 다시 시작
- **정답 안내**: 그리드 위에 반투명 정답 미리보기 + 2D 이미지 (TAB으로 켜고 끄기)
- **정답 여러 개 넣으면 라운드마다 랜덤**으로 하나 나옴
- 등록한 블록이 **인게임에서 진짜 모양(프리팹)으로** 보임

작업하는 씬이 둘로 나뉘어요:

| 씬 | 여는 법 | 여기서 하는 일 |
|---|---|---|

## 1. 크기 규칙 (PPU부터 정리할게요)

- **칸 1개 = Unity 월드 1유닛.** 그리드는 항상 이 기준이에요.
- 블록 크기는 **footprint(차지하는 칸 수)** 로 정해요:
  - `(1,1,1)` = 한 칸짜리 큐브
  - `(1,1,3)` = z 방향으로 3칸
  - `(1,3,2)` = 벽처럼 생긴 것
- **PPU 128은 "텍스처 화질" 기준일 뿐, 크기랑은 무관해요.**
  - 뜻: 텍스처를 **1유닛당 약 128픽셀** 밀도로 그린다(로우폴리·카툰 톤 통일).
  - 3D 메시는 임포트 PPU 설정이 아니라 **UV를 그 밀도로** 맞추면 돼요. (UI용 2D 스프라이트가 있다면 그건 임포트 PPU=128)
  - 👉 **블록 크기·배치·채점은 footprint(칸 수)로만 정해져요. PPU는 영향 없어요.**

## 2. 블록(프리팹) 만들 때 규약

VARCO AI로 만든 3D 메시를 Unity로 가져와 프리팹으로 만들 때, 아래 4가지만 지켜주세요:

1. **스케일** — 칸=1유닛에 맞춰요. footprint `(fx,fy,fz)` 블록이면 화면에서 `fx×fy×fz` 유닛 크기.
2. **피벗(원점) = footprint의 min-corner** (가장 작은 x,y,z 모서리). 메시가 **+X/+Y/+Z 방향으로 뻗게** 두면 회전·확장이 딱 맞아요.
3. **콜라이더** — 보이는 모양에 맞춰 넣어요 (플레이어 충돌용).
4. **아트** — PPU 128 / 로우폴리 카툰 톤.

> 💡 1×1×1 큐브든 기둥/벽 같은 멀티칸 메시든, **footprint 숫자만 다를 뿐 똑같이** 처리돼요.

## 3. 블록 등록 — 3D 모델을 게임에 "연결"하기 ⭐

블록 하나를 게임에 넣는 건 **4단계 체인**이에요. 한 단계가 다음으로 이어져요:

```
① 3D 모델 → 프리팹           (게임에 보이고 부딪힐 진짜 물건)
        ↓  "이게 무슨 블록인지" 적은 카드를 만들고
② MaterialDef 에셋           (번호·크기·프리팹·공정 = 블록 설명 카드)
        ↓  게임이 아는 "블록 목록"에 카드를 넣고
③ MaterialCatalog            (모든 블록의 마스터 목록)
        ↓  맵 칠할 때 쓰는 "팔레트"를 갱신하면
④ Autotiles3D 타일           (메뉴 한 번이면 자동!)
```

### ① 프리팹 만들기
- **[Project]** VARCO 3D 모델을 가져와 §2 규약대로 **프리팹**으로 만들어요.
- *왜?* → 게임 화면에 띄우고 충돌 판정할 진짜 오브젝트라서요.

### ② MaterialDef = 블록 "설명 카드" 만들기
- **[Project]** 빈 곳 **우클릭 → Create → Grid → MaterialDef**, 이름 지어요 (예: `Mat_Window`).
- 만든 카드를 누르고 **[Inspector]** 에서 채워요:

  | 칸 | 뜻 | 예시 |
  |---|---|---|
  | **Id** | 고유 번호 (다른 재료와 안 겹치게) | `3` |
  | **Footprint** | 차지하는 칸 (x,y,z) | `(1,2,1)` |
  | **Prefab** | ①에서 만든 프리팹 | Window 프리팹 |
  | **RequiredProcesses** | 필요한 공정 (없으면 비움) | `Fixed` |
  | **MustBeFixed** | 하중 부재(기둥/벽)면 체크 → 안 고치면 충격에 무너짐 | ✔ |
  | IsBreakable / MaxSpawnCount | 유리 등 / 스폰 제한(-1=무제한) | |

- *왜?* → "이 블록은 3번, 1×2×1 크기, 망치질 필요" 같은 정보를 게임이 읽어요.

### ③ MaterialCatalog에 카드 넣기
- **[Project]** `Assets/Grid/Data/MaterialCatalog`를 눌러요.
- **[Inspector]** 의 **Materials** 목록에 ②에서 만든 카드를 **끌어다 놓아요.**
- *왜?* → 게임은 이 목록으로 "세상에 있는 블록 전부"를 알아요.

### ④ 맵 팔레트(타일) 갱신하기
- **[메뉴] Grid Setup → Create Autotiles3D Tiles From Catalog** 를 한 번 눌러요.
- → 카탈로그의 모든 재료로 타일이 자동 생성돼요 (타일 이름 = 재료 이름, 프리팹은 ①의 것).
- *왜?* → 맵 만들 때 이 블록을 골라서 칠할 수 있게요.

✅ **끝! 코드는 한 줄도 안 건드려요** — 전부 에셋과 메뉴로 끝나요.
(샘플 블록 Floor/Pillar/Wall은 이미 ①~④가 다 돼 있어요.)

## 4. 맵(정답) 만들기 — Autotiles3D

🗺️ **씬: AnswerAuthoring.unity** (Project → `Assets/Grid/Scenes` → 더블클릭)

1. **[메뉴] Grid Setup → ★ Setup Autotiles3D Authoring**
   → 타일·그리드·레이어가 자동으로 세팅돼요. **[Hierarchy]** 에서 `Answer Layer`가 자동 선택돼요.
2. **[Inspector]** 에 타일 썸네일(Mat_Floor/Pillar/Wall…)이 떠요 → **하나 클릭**해서 골라요.
3. **[Scene]** 그리드 위에 마우스를 올리면 미리보기가 떠요:
   - **좌클릭** = 칠하기 · **우클릭** = 지우기 · **Ctrl+휠** = 회전 · **Alt+휠** = 층(Y) 이동
   - 정답 모양대로 칸칸이 칠해요.
4. **[메뉴] Grid Setup → Export Answer from Autotiles3D**
   → `Assets/Grid/Data/ExportedAnswer.asset` 이 생겨요. **[Console]** 에 "익스포트 N칸" 로그가 떠요.

> 💡 채점은 **칸별로 재료가 맞는지** 봐요. 그래서 멀티칸 재료는 그 칸들을 칠하면 인게임 배치와 자동으로 맞아요. **회전은 신경 안 써도** 돼요 (위치로만 판정).

## 5. 게임에 적용 + 정답 여러 개(랜덤)

🎮 **씬: 게임 씬** (GridManager가 있는 씬)

1. **[Hierarchy]** 에서 **GridManager**를 골라요.
2. **[Inspector]** 의 **Answers** 목록에 정답 에셋(ExportedAnswer 등)을 **끌어다 놓아요.**
3. **여러 개 넣으면** 게임 시작/재시작마다 **서버가 랜덤으로 하나**를 골라 모두에게 똑같이 줘요. 한 개면 항상 그거예요.
4. Play → 정답 미리보기가 칠한 모양으로 뜨면 성공! 🎉

## 7. 조작키 & 왜 이렇게 바뀌었나

> ⚠️ **조작 키는 "테스트하면서 임시로" 정한 거예요.** 불편하거나 다른 키가 낫겠다 싶으면 **말씀만 주세요 — 바로 바꿔드려요.** 아직 구현 안 한 것도 더 하면 되고요.
>
> ✅ **공정(고정·페인트)은 원래 기획대로 "도구 들고 와서 E 꾹 → 로딩바" 방식이에요.** ① 작업장(망치=고정 / 페인트통=페인트)에 다가가 **F로 도구를 들고** → ② 블록을 마우스로 가리키고 **E를 꾹** → 로딩바가 차면 그 공정이 적용돼요. **든 도구가 그 블록에 지금 필요한 공정과 맞을 때만** 바가 차요(예: 페인트는 고정부터 끝난 블록에만). 어떤 블록에 뭐가 필요한지는 **TAB**(정답 안내)으로 색을 보고 확인해요 — 회색=배치만, 파랑=고정, 초록=페인트.

**인게임 조작 요약**
> 이동 `WASD` · 마우스로 칸 조준 · **좌클릭** 배치 · **우클릭** 철거 · **F** 줍기/도구 · **E(꾹)** 공정(든 도구 적용) · **G** 던지기(조준 방향·협동 전달) · **R** 회전 · **마우스휠** 층 · **Space** 버리기(바닥에) · **C** 공정취소 · **TAB** 정답/필요공정 · **Enter** 건축종료 동의/재시작

**기존 기획에서 바꾼 것과 이유** (테스트하며 더 나은 쪽으로 조정한 것들):

| 항목 | 원래 기획 | 지금 | 바꾼 이유 |
|---|---|---|---|
| **블록 배치** | 플레이어 앞에 Space로 조준해서 놓기 | **마우스로 가리킨 칸에 좌클릭** | Space 조준이 너무 어려웠어요 |
| **재료 얻기** | 숫자키(1/2/3) 팔레트에서 선택 | **주문 → 배송 → 다가가 F로 줍기** (리썰컴퍼니식) | 재료 10개 넘으면 숫자키가 의미 없고, 한 번에 하나만 들 수 있어서요 |
| **재료 줍기** | 가장 가까운 것 자동 | **마우스로 가리킨 것**(+손 닿는 범위) | 가까운 것만 집으면 원하는 걸 못 집어서요 |
| **공정(고정/페인트)** | 도구 들고 좌클릭 | **도구 들고 블록 가리켜 E 꾹 → 로딩바** | 원래 기획대로(로딩바). 든 도구가 맞는 공정일 때만 적용 |
| **건축 층 이동** | `Q`/`E` | **마우스휠** | E를 공정에 내줘서요 |
| **버리기/철거/붕괴** | 사라짐 | **바닥에 떨어져 다시 주움** | 자원을 잃지 않게 (물류창고 일관성) |
| **노답중력** | 떨어지는 연출만 | **몸에 닿으면 발로 차여 굴러감 + 원래 크기 유지** | 무빙아웃식 물리 재미 |
| **정답 판정** | — | **절대 위치**(기획서대로), 회전은 칸으로 판정 | 같은 모양 대칭 블록이 회전값만 달라 감점되는 걸 막으려고 |
