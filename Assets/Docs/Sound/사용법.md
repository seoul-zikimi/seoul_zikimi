# SoundManager 사용설명서

SoundManager는 게임의 모든 BGM과 SFX를 한 곳에서 관리하는 매니저입니다.  
BGM은 게임 상황(페이즈)에 따라 자동으로 부드럽게 전환되고,  
SFX는 AudioClip을 직접 들고 다닐 필요 없이 enum 타입 하나만으로 호출할 수 있습니다.

## 1. 구조 한눈에 보기

### 핵심 3가지

**BGM** — 게임 상황이 바뀔 때 `SetPhase()`로 음악 전환
```csharp
SoundManager.Instance.SetPhase(GamePhase.Building);       // 건축 시작
SoundManager.Instance.SetPhase(GamePhase.BuildingUrgent); // 타이머 30초 남음
```

**SFX 2D** — 거리와 무관한 소리
```csharp
SoundManager.Instance.PlaySFX(SFXType.PlayerFootstep);
SoundManager.Instance.PlaySFX(SFXType.PlayerBounce);
```

**SFX 3D** — 월드 공간에서 들려야 하는 소리 (멀리 있으면 작게 들림)
```csharp
// 예: 블록 무너짐 위치에서 소리
SoundManager.Instance.PlaySFXAt(SFXType.BlockCollapse, transform.position);
```

### 어떻게 클립을 관리하나요?

기존 QuackToHell은 각 스크립트마다 `[SerializeField] AudioClip sfx` 필드를 들고 다녔습니다.  
이 방식은 프리팹이 많아질수록 연결 관리가 힘들어집니다.

SoundManager는 **`SoundLibrary.asset` 하나**에 모든 클립을 모아둡니다.  
어떤 스크립트든 `SoundManager.Instance.PlaySFX(SFXType.PlayerFootstep)` 한 줄이면 됩니다.  
클립을 바꾸고 싶으면 `SoundLibrary.asset`만 열어서 교체하면 됩니다.

### SFX는 어떻게 재생되나요?

- **2D** (`PlaySFX`): SoundManager의 AudioSource **1개**에 `PlayOneShot`. 한 소스로 여러 소리가 겹쳐 재생됩니다 (`new GameObject()` 없음 → GC 부하 없음).
- **3D** (`PlaySFXAt`): 위치마다 소리가 따로 들려야 하므로 AudioSource를 `_sfx3DVoices`개(기본 8) 미리 만들어 두고 round-robin으로 돌려씁니다. 시작 시 1번만 생성하므로 GC 부하가 없습니다.
  - 동시 발음이 voice 수를 넘으면 가장 오래된 소스를 재사용합니다(그 소리는 새 위치를 따라감). 동시 3D가 많으면 `_sfx3DVoices`를 올리세요.

> enum 없이 잠깐 쓸 클립은 `PlaySFX(AudioClip)` / `PlayBGM(AudioClip)` 오버로드로 등록 없이 바로 재생할 수 있습니다.

## 2. BootstrapScene에 배치하기

SoundManager는 UIManager와 동일한 방식으로 **BootstrapScene에 한 번만** 배치합니다.

### 배치 순서

**① BootstrapScene을 엽니다.**

**② SoundManager 오브젝트를 만듭니다.**
```
Hierarchy 빈 곳 우클릭 → Create Empty → 이름을 "@SoundManager"로 변경
```

**③ SoundManager 스크립트를 붙입니다.**
```
Inspector → Add Component → "SoundManager" 검색 → 선택
```

**④ 필드 3개를 연결합니다.**

| 필드 | 연결할 에셋 | 위치 |
|------|-----------|------|
| `_library` | `SoundLibrary.asset` | `Assets/Data/Sound/` |
| `_mixer` | `GameAudioMixer.mixer` | `Assets/Data/Sound/` |
| `_bgmFadeDuration` | 기본값 1 (초) | BGM crossfade 길이 |
| `_sfx3DVoices` | 기본값 8 | 동시 3D 효과음 최대 개수 |

**⑤ 완성된 BootstrapScene Hierarchy:**
```
BootstrapScene
├── @UIManager         ← UIManager.cs 붙어있음
├── @SoundManager      ← SoundManager.cs + 필드 연결
└── EventSystem
```

> ⚠️ **SoundLibrary.asset에 클립을 연결하지 않으면 소리가 나지 않습니다.**  
> 클립 파일이 아직 없다면, 일단 빈 배열로 두고 나중에 연결해도 됩니다.  
> SFXType이 등록되지 않은 경우 조용히 넘어갑니다 (에러 없음).

## 3. AudioMixer 세팅

SoundManager는 `GameAudioMixer.mixer`가 필요합니다. **에디터에서 직접 생성**해야 합니다.

### 생성 순서

**① AudioMixer 파일 생성**
```
Project 창 → Assets/Data/Sound/ 폴더에서 우클릭
→ Create → Audio Mixer → 이름을 "GameAudioMixer"로 변경
```

**② BGM / SFX 그룹 추가**
```
AudioMixer 창 열기 (더블클릭)
→ Groups 패널에서 "Master" 선택 → 하단 "+" 클릭
→ "BGM" 그룹 생성
→ "Master" 다시 선택 → "+" 클릭 → "SFX" 그룹 생성
```

**③ BGMVolume 파라미터 Expose**
```
BGM 그룹 선택 → Inspector에서 Volume 슬라이더 우클릭
→ "Expose 'Volume (of BGM)' to script" 클릭
→ AudioMixer 창 상단 "Exposed Parameters" 클릭
→ 파라미터 이름을 정확히 "BGMVolume"으로 변경
```

**④ SFXVolume 파라미터 Expose**
```
SFX 그룹 선택 → Inspector에서 Volume 슬라이더 우클릭
→ "Expose 'Volume (of SFX)' to script" 클릭
→ Exposed Parameters에서 이름을 "SFXVolume"으로 변경
```

> ⚠️ Exposed Parameter 이름이 정확히 `BGMVolume`, `SFXVolume`이어야 합니다.  
> 오타가 있으면 볼륨 설정이 동작하지 않습니다.

### SoundLibrary.asset 생성

```
Project 창 → Assets/Data/Sound/ 에서 우클릭
→ Create → Sound → Library → 이름을 "SoundLibrary"로 변경
```

Inspector에서 SFX/BGM 클립을 연결합니다.

## 4. BGM 페이즈 전환하기

### GamePhase 목록

| Phase | 언제 | BGM 분위기 |
|-------|------|-----------|
| `Lobby` | 로비 / 팀 메이킹 | 느긋함 |
| `Building` | 건축 타이머 시작 | 활기참 |
| `BuildingUrgent` | 타이머 30초 이하 | 긴박함 |
| `Result` | 채점 / 결과 화면 | 성취감 |

### 호출 방법

```csharp
// 게임 시작 시 (타이머 매니저 또는 게임흐름 매니저에서)
SoundManager.Instance.SetPhase(GamePhase.Building);

// 타이머 30초 이하 감지 시
SoundManager.Instance.SetPhase(GamePhase.BuildingUrgent);

// 건축 종료 / 타이머 0 시
SoundManager.Instance.SetPhase(GamePhase.Result);

// 로비로 돌아올 때
SoundManager.Instance.SetPhase(GamePhase.Lobby);
```

### BGM이 전환되는 방식

BGM이 즉시 바뀌지 않고, 1초에 걸쳐 부드럽게 페이드 아웃 → 페이드 인됩니다.  
같은 Phase를 두 번 호출해도 안전합니다. 이미 같은 클립이 재생 중이면 아무것도 하지 않습니다.

### BGM 정지

```csharp
SoundManager.Instance.StopBGM();      // 즉시 정지
SoundManager.Instance.StopBGMFade();  // fade-out 후 정지
```

> 🔧 **아직 타이머/게임흐름 매니저가 없습니다.**  
> `SetPhase()` 호출은 해당 매니저가 구현된 후 연결합니다.  
> 그 전에는 `SoundManagerDemo.cs`로 수동 테스트할 수 있습니다.

## 5. SFX 호출하기

### 기본 원칙: AudioClip 필드를 스크립트에 두지 마세요

**❌ 기존 방식 (하지 마세요)**
```csharp
[SerializeField] AudioClip _pickUpClip;  // 프리팹마다 연결 관리 필요

void TryPickUp() {
    SoundManager.Instance.SFXPlay("PickUp", _pickUpClip);
}
```

**✅ 새 방식**
```csharp
// AudioClip 필드 없음, enum만 사용
void OnCollisionEnter(Collision collision)
{
    // ...기존 bounce 로직...
    SoundManager.Instance.PlaySFX(SFXType.PlayerBounce);
}
```

### 현재 연결된 SFX

| 타입 | 호출 위치 | 동작 |
|------|----------|------|
| `PlayerFootstep` | `PlayerDustTrail.Update()` | 이동 중 0.35초마다 1회 |
| `PlayerBounce` | `PlayerBounce.OnCollisionEnter()` | 플레이어끼리 충돌 시 |

### SFXType에 새 항목 추가하는 법

1. `Assets/Scripts/Sound/SFXType.cs`에 enum 값 추가
2. `SoundLibrary.asset` Inspector에서 SFX Entries에 항목 추가 → 클립 연결
3. 호출 위치에서 `SoundManager.Instance.PlaySFX(SFXType.새항목)` 추가

### 예측: 추후 시스템 구현 시 추가될 SFX

> 아래는 예측입니다. 해당 시스템 구현 시점에 SFXType enum에 추가하고 SoundLibrary.asset에 클립 연결.

| 타입 (예측) | 연결 예정 위치 | 조건 |
|-----------|-------------|------|
| `BlockPickUp` | `PlayerBuildingHandler.TryPickUp()` | 건축 시스템 구현 후 |
| `BlockPlace` | `PlaceBuildingCommand.Execute()` | 건축 시스템 구현 후 |
| `BlockFixed` | `GridManager.TryAdvanceProcess()` | Fixed 상태 전환 시 |
| `BlockFinished` | `GridManager.TryAdvanceProcess()` | Finished 상태 전환 시 |
| `BlockCollapse` | `CollapseManager` [ClientRpc] | 붕괴 시스템 완성 후 |
| `TimerWarning` | 타이머 매니저 | 타이머 구현 후 |
| `ButtonClick` | 각 HUD / Popup | UI 구현 후 |

### 멀티플레이어 규칙

로컬 플레이어 액션은 `if (IsOwner)` 안에서, 서버 이벤트는 `[ClientRpc]`로 전파:

```csharp
// 로컬 액션 — 자기 화면에서만
if (IsOwner) SoundManager.Instance.PlaySFX(SFXType.BlockPickUp);

// 서버 이벤트 — 모든 클라이언트에 전파
// NGO 규칙: [ClientRpc] 어트리뷰트 필수 + 메서드명 반드시 ClientRpc 로 끝나야 함
[ClientRpc]
void PlayCollapseClientRpc(Vector3 pos)
{
    SoundManager.Instance.PlaySFXAt(SFXType.BlockCollapse, pos);
}
// 서버 측 호출: PlayCollapseClientRpc(transform.position);
```

## 6. 볼륨 슬라이더 연결하기

설정 UI의 슬라이더(0~1 범위)를 `OnValueChanged`에 연결합니다.

```csharp
// SettingsHUD.cs — Init() 또는 Start() 내부
bgmSlider.onValueChanged.AddListener(val =>
    SoundManager.Instance.SetBGMVolume(val));

sfxSlider.onValueChanged.AddListener(val =>
    SoundManager.Instance.SetSFXVolume(val));
```

볼륨은 PlayerPrefs에 자동 저장됩니다. 게임을 껐다 켜도 마지막 설정이 복원됩니다.

## 7. 데모씬으로 테스트하기

### 전제 조건 (먼저 완료)

1. `GameAudioMixer.mixer` 생성 + BGM/SFX 그룹 + Exposed Parameters 설정 (3번 참고)
2. `SoundLibrary.asset` 생성 + 임시 클립 연결

### 씬 생성

```
Unity 메뉴 → Game > Setup > Create SoundManager Demo Scene
```

`Assets/Scenes/SoundManagerDemoScene.unity` 생성됨.  
`@SoundManager`의 `_mixer`, `_library` 필드가 자동 연결됨 (에셋이 미리 있을 때).

### 키보드 단축키

| 키 | 동작 |
|----|------|
| `[1]` | BGM: Lobby |
| `[2]` | BGM: Building |
| `[3]` | BGM: BuildingUrgent |
| `[4]` | BGM: Result |
| `[Q]` | 2D SFX: PlayerFootstep |
| `[W]` | 2D SFX: PlayerBounce |
| `[A]` / `[D]` | 3D SFX: 왼쪽 / 오른쪽 (위치 패닝) |
| `[S]` | 3D SFX 버스트 — 8곳 동시 발음 (voice 풀) |
| `[Z]` / `[X]` | SFX 볼륨 − / + |
| `[C]` / `[V]` | BGM 볼륨 − / + |
| `[F1]` | BGM 즉시 정지 |
| `[F2]` | BGM fade 정지 |
| `[F3]` | SFX 전체 정지 (2D + 3D) |

### 확인 항목

- [ ] `[Q]` 누름 → 2D SFX 재생
- [ ] `[2]` 누름 → BGM 재생, `[3]` 누름 → 1초 crossfade로 전환
- [ ] `[A]`/`[D]` → 왼쪽/오른쪽에서 들림(3D), `[S]` → 8곳에서 퍼져 들림(voice 풀)
- [ ] `[S]` 직후 `[Z]` 연타 → 울리던 3D 소리도 같이 작아짐(3D 믹서 볼륨 적용)
- [ ] `[F1]` BGM 즉시 정지, `[F2]` fade 정지, `[F3]` 2D·3D SFX 모두 정지
- [ ] Unity Profiler → GC Alloc 0 확인 (SFX 연속 재생 시)

## 8. [참고] 호출 위치 정리

### ✅ 지금 연결된 것

```csharp
// PlayerDustTrail.Update() — 이미 연결됨
if (isMoving)
{
    m_FootstepTimer -= Time.deltaTime;
    if (m_FootstepTimer <= 0f)
    {
        SoundManager.Instance.PlaySFX(SFXType.PlayerFootstep);
        m_FootstepTimer = 0.35f;
    }
}
else m_FootstepTimer = 0f;

// PlayerBounce.OnCollisionEnter() — 이미 연결됨
SoundManager.Instance.PlaySFX(SFXType.PlayerBounce);
```

SoundManager.Instance.PlaySFX(SFXType.BlockPlace);

if (newState == BuildState.Fixed)    SoundManager.Instance.PlaySFX(SFXType.BlockFixed);

{
    SoundManager.Instance.PlaySFXAt(SFXType.BlockCollapse, pos);
}

## 9. [참고] QuackToHell과 무엇이 다른가

| 항목 | QuackToHell (Rokiss) | 서울지키미 |
|-----|---------------------|---------|
| BGM 트리거 | `scene.name == clip.name` (씬이름 하드코딩) | `GamePhase` enum (상황 기반) |
| BGM 전환 | 즉시 교체 | 1초 crossfade |
| SFX 호출 | `SFXPlay(name, clip)` — 호출자가 clip 소유 | `PlaySFX(SFXType)` — SoundManager가 clip 소유 |
| SFX GC | `new GameObject` per SFX + `Destroy` | 2D 1소스 PlayOneShot + 3D voice 풀 재사용, GC Alloc 없음 |
| 볼륨 저장 | 저장 안 함 | PlayerPrefs 자동 저장/복원 |
| 3D 공간음 | 없음 | `PlaySFXAt(type, pos)` |
